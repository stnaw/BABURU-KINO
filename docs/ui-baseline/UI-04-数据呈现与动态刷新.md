# 数据呈现与动态刷新

## 当前数据分层

当前前端存在三类数据源：

1. 链上只读数据
2. 钱包相关数据
3. 本地开发模拟数据

其中价格位属于第 3 类，本地开发下由 QA 脚本生成。

## 指标卡动画

当前概览数字采用统一的数值补间方案：

- 持续时间：`860ms`
- easing：`1 - (1 - progress)^5`
- 逐帧渲染

这套动画已用于：

- 可借 BNB
- 金库 BNB 总量
- BABURU 质押
- BABURU 已销毁
- BABURU 待清算
- 活跃借款数
- 单笔可借比例
- BABURU 价格

## 价格位

当前价格位不是链上真实预言机价格，而是本地开发模拟价格。

现状实现：

- `tax` 模拟脚本会把最新价格写入 `artifacts/qa-logs/latest-market.json`
- 前端定时读取该文件
- 页面只更新价格这一格，不再触发整页热重载

这套设计是为了兼顾两个目标：

- 本地开发时有“活”的价格
- 不破坏页面整体稳定性

## 借款估值刷新

借款区当前有独立刷新元信息，表现为：

- 圆环刷新提示
- 自动倒计时驱动
- 刷新后估值数字重新播放动画

当前这套刷新只服务于借款估值，不等于整页链上数据刷新。

## 开发者面板的数据刷新策略

开发者面板的数据分为两类：

- 主页面本来就需要的数据
- 只有开发者面板需要的数据

当前策略是：

- 公共数据随全局只读刷新更新
- 面板专属数据仅在面板打开时刷新

这样可以避免 owner 专属信息持续占用不必要的刷新成本。

## 大数字格式

当前指标数字格式有两个明显规则：

- 大于等于百万时自动切成 `M`
- 超小小数支持下标零位展示

并且为了视觉统一：

- 即使没有 `M` 单位，也会预留单位空间

这让不同量级的数字在卡片右侧更容易对齐。
