# 前端展示（UI 交互规格 · 对齐金库合约）

> 链：**BSC** · 本币：**BABURU** · 金库：**BABURU KINKO**  
> 规则细则以 `Design/税率分配.md` 为准；本文从**视觉基调、页面、流程、控件、状态**描述产品交互。
> Web 页面包含电脑端 和 移动端。
---

## 1. 设计风格与品牌

| 维度 | 说明 |
|------|------|
| **品牌** | 项目名 **BABURU**；叙事与视觉可向 **bubble（气泡）** 联想：轻盈、圆润、偏可爱，与「金库」形成轻松记忆点。 |
| **整体气质** | **可爱风**为主。涉及金额、倒计时、风险状态时，**数字与标签**仍需高对比、易扫读；手绘感侧重**卡片外框、按钮、Tab、弹窗、插画**，避免关键数据「糊」在装饰里。 |
| **组件基线** | 控件形态参考开源手绘风组件集 [sketchbook-ui](https://github.com/SarthakRawat-1/sketchbook-ui)（笔触描边、柔和填充、松弛圆角等）。若栈不兼容，需自建 **与 Sketchbook 气质一致** 的设计 token（描边、主辅色、纸/涂鸦背景），并在设计稿中写死，保证全站统一。 |
| **插画与空状态** | 空列表、错误页、帮助入口等可配与气泡/金库主题一致的轻量插画，与 §6.5、帮助入口联动。 |
| **可访问性** | 可爱风不降低 a11y：**最小点击区域**、**焦点可见**、状态色不仅依赖色相（成功/警告/错误需图标或文案辅助）。 |

## 3. 信息架构与导航

建议主结构：**顶栏 + Tab 或锚点分区**（单页滚动亦可，但借款/还款建议视觉分区清晰）。

| 导航项 | 内容 |
|--------|------|
| **概览** | 金库与市场信息、整理金库入口、试算、合约状态横幅 |
| **借款** | 质押输入 → 预估 → 授权/下单（与合约一步或多步一致） |
| **我的借款** | 未结清订单列表、筛选/多选、还款确认 |
| **金库手册** | 内嵌 FAQ 或侧栏/弹层打开 FAQ |
| **透明度**（可选） | 只读：ρ、黑名单列表、合约地址与链接 |

**顶栏常驻**

- Logo / 产品名  
- 网络指示：**BSC**；错误网络时 **切换网络** CTA（主按钮态）  
- **连接钱包** / 已连接则展示**缩略地址** + **断开**（次要操作）  
- 已连接时可在顶栏或概览区展示 **BNB**、**BABURU** 余额（小字或图标 Tooltip：「还本用 / 质押用」）

## 4. 概览区（仪表盘）

布局建议：**状态横幅（若有）→ 关键指标卡片栅格 → 试算模块 → 次要指标**。

### 4.1 全局状态横幅（条件展示）

- **新借款已暂停**：醒目条（Warning），文案说明「仅暂停**新开借款**；**还款与清算不受影响**」，避免用户误以为资金锁死。  
- **只读展示 ρ**（若有链上值）：可与横幅或某张卡片合并，避免重复占位。

### 4.2 指标卡片（建议桌面端单行并排、移动端回落）

| 卡片 | 展示内容 | 交互 |
|------|----------|------|
| 金库 BNB | 实时余额（对齐合约 `address(this).balance` 语义） | 可选：刷新按钮或自动轮询间隔说明 |
| 单笔可借比例 | 当前链上 `ρ` 对应的单笔可借比例 | 建议与 BNB 库存并列展示 |
| 活动敞口 | 未结清笔数 | Tooltip 链到「可借分母」说明（FAQ / 小字） |
| BABURU 复合卡 | `BABURU` 主标题、价格位（未发行时可为 `TBA`）、**已质押 BABURU**、**待清算 BABURU** | 同卡内提供 **整理金库** 按钮 |
| 试算锚点 | 「质押 **1M** BABURU 约可借 **x.xxx BNB**」 | 旁注：**非成交保证**、随区块变化 |

试算公式与分母定义须与 `税率分配.md` §5 一致；卡片下方一行灰色说明：**数值为当前链上快照**（样式遵循 §1 手绘卡片基线）。

### 4.3 未连接态

- 指标仍可展示（公共数据），**借款 / 我的借款** 内 CTA 为「连接钱包」或禁用并 Tooltip「请先连接」。

## 5. 借款流程（交互分步）

### 5.1 布局

- 左侧（或上部）：**质押数量**输入（支持 MAX = 钱包 BABURU 可转余额）。  
- 右侧（或下部）：**预估可借 BNB**（`refBorrow`）、**最小可借比例**控件、主 CTA 区。

### 5.2 控件细则

| 控件 | 行为 |
|------|------|
| 质押输入 | 实时校验：>0、≤余额；输入变化时**防抖**后重算 `refBorrow`，避免 RPC 轰炸 |
| 预估可借 | 大字展示 BNB；小字：「提交时以链上为准」 |
| **最小可借比例** | **滑杆** + 可选数字输入（手绘风轨道/拇指与 §1 一致）；默认 **95%**（9500 bps），范围 **0.01%～100%**（1～10000 bps）；旁 **?** 说明：低于此比例则交易回滚（对应 `currBorrow` 与 `refBorrow × minBorrowBps` 校验） |
| 主 CTA | 建议合并成**单个主按钮**：授权不足时显示 **授权并借款**，授权已足够时显示 **确认借款** |

### 5.3 授权与交易状态机

1. 用户点主按钮前：若 `allowance` 不足，按钮先完成 **Approve**，授权成功后继续借款；若授权已足够，则直接提交借款。  
2. 交易发送后：**Pending**（哈希可点浏览器）→ **Success**（ toast + 引导去看「我的借款」）→ **Failed**（见 §8 错误映射）。  
3. Success 后：**清空输入或保留**由产品定；**列表自动刷新**。

补充建议：

- 借款面板可显示**全局刷新倒计时**与圆形进度，提示下一次链上库存与试算同步时间。

### 5.4 禁用与拦截（借款暂停 / 分母异常）

- **全局暂停新开借**：借款 Tab 内主 CTA **禁用**，与概览横幅一致文案。  
- **分母 ≤ 0** 或试算失败：主 CTA 禁用 + 简短说明「当前不可借」。  
- **质押为 0**：主 CTA 禁用。

## 6. 我的借款 / 还款流程

### 6.1 列表

- **排序**：默认 **时间倒序**（最新在上）。  
- **行内容**（每条订单）：质押 BABURU、借出 BNB、**T0**、**当前阶段标签**（提前 / 正常 / 宽限 / **已超期·不可还**）；若后续信息密度允许，可补充 D+6 / D+9 关键日提示。  
- **视觉**：宽限、超期可用色条或标签区分；**已满 9 天**行：**勾选框禁用**或隐藏勾选，附文案「待清算 / 仅清算路径」。

### 6.2 多选与底部汇总条（Sticky）

- 表头或首列：**全选**（仅针对**可选**订单）。  
- 选中 ≥1 时，底部固定条展示：  
  - **应还 BNB 合计**  
  - **罚金 BABURU 合计**（或展开「逐笔明细」折叠面板）  
  - 主 CTA：**确认还款**

### 6.3 确认弹窗（模态）

- 逐笔：借出 BNB、本笔应还 BNB、罚金比例与罚金数量、阶段说明。  
- 合计：BNB 总额、BABURU 罚金总额。  
- 明确提示：**请在钱包中支付与合计一致的 BNB**（与合约「总额不一致则 revert」一致）。  
- 二次确认按钮：**在钱包中确认** / **取消**。

### 6.4 提交前客户端校验（减少 revert）

- **BNB**：钱包 BNB ≥ 应还合计（可预留 gas 提示）。  
- 还款时不再要求钱包额外持有 BABURU 支付罚金；罚金会直接从订单质押中扣除。  
- 不满足：仅保留 **BNB** 与 gas 提示。

### 6.5 空状态

- 无未结清订单：**§1 风格**下的轻量插图或简洁文案 + 引导至「借款」Tab。

## 8. 反馈、加载与错误映射

| 类型 | 交互建议 |
|------|----------|
| 读链 | 骨架屏或卡片内 Spinner；失败时 **重试** |
| 写链 | 按钮 Loading；禁止重复点击；成功 Toast + 列表刷新 |
| 合约 Revert | 映射为用户可读短句 + 「详情见 FAQ」链接，例如：可借不足 / 分母无效 / BNB 与订单总额不符 / 非本人订单 / 暂停借款 等 |

## 9. FAQ（项目规则速览）

面向用户；细则以 `Design/税率分配.md` 为准。实现上建议：**帮助图标 / 页脚 / 抽屉** 与下文同内容。

**Q1：BABURU 和金库是什么关系？**  
A：BABURU 是运行在 **BSC** 上的本币；**BABURU KINKO** 是金库合约。买卖税换成的 BNB 会进入金库；你可以在金库**质押 BABURU** 借出 **BNB**，到期前/宽限内还 BNB，并按规则从质押中扣除罚金后解押剩余 BABURU。

**Q2：买卖税多少？税金去哪？**  
A：买入 **2%**、卖出 **4%**（先收本币再换 BNB）。换得的 BNB **100% 进金库**；金库不会主动去「抽税」，而是按规则接收入账。

**Q3：一笔借款的时间线怎么算？**  
A：以链上**借款成功时刻 T0** 为准。**正常借款期 6 天**；接着 **3 天宽限期 [6, 9) 天**——仍可还，但要交**滞纳类罚金**。**满 9 天及以上**视为超期：**不能再按还款解押**，质押本币会走**清算销毁路径**（本币 `transfer` 至 `0x…dead`；与 `burn()` 不同，一般不减少 `totalSupply`）。

**Q4：什么时候还 BNB 要交罚金？交多少？**  
A：罚金按**每笔订单、当前所处时间段**单独算：`罚金 = 该笔质押 BABURU × 档位比例`。罚金不会额外从你的钱包收取，而是直接从该笔已质押 BABURU 中扣除，并统一按项目约定的销毁口径处理。  
- **提前还**（T0 后 0～3 天）：罚金比例依次为 **90% → 60% → 30%**（按天分段）。  
- **正常还**（**[3, 6) 天**）：**0%** 罚金。  
- **宽限还**（**[6, 9) 天**）：滞纳罚金 **30% → 60% → 90%**（按天分段）。  
- **超期（≥9 天）**：不能还款解押；质押 **100%** 进入销毁路径。

**Q5：我能借多少 BNB？为什么试算和实际不一致？**  
A：单笔可借（在池子允许的前提下）大致为：  
`可借 BNB = 本笔质押 / 可借份额分母 × (金库当时 BNB 规模 × ρ)`，默认可借比例 **ρ = 0.7**（链上可调）。  
分母 = **初始总量常量 − 黑名单地址持仓本币 − 金库内未结清质押**。别人借款、黑名单地址持仓变化、金库 BNB 增减都会让**同一质押数量对应的可借 BNB 实时变化**；前端试算只是快照参考。  
若分母 **≤ 0**，该笔借款会直接失败。

**Q6：什么是「最小可借比例」？为什么交易失败？**  
A：你可设置 **minBorrowBps**（如默认 95%）：只有合约算出的实时可借 **不低于** 你看到的参考值 × 该比例时，借款才会成功；否则整笔回滚，防止价格波动导致「借到的远少于预期」。

**Q7：可以借多笔吗？可以一起还吗？**  
A：可以**多笔借款**；还款支持**勾选多笔订单批量处理**。注意：你传入的 **BNB 总额必须与合约算出的应还 BNB 总额一致**，否则回滚。

**Q8：满 9 天的订单在页面上怎么处理？**  
A：这类订单**不应作为普通还款勾选**；若误传，合约侧会按规则处理销毁且**不计入应还 BNB**。任何人都可以发起**整理金库**，把这些已超期借款推进到链上结清与销毁流程。

**Q9：项目方能把金库里的 BNB 转走吗？**  
A：设计目标是：**金库 BNB 不得被任意地址随意转出**；仅允许白名单路径（如税金入账、借款、还款回流等）。具体权限（治理调 ρ、黑名单、暂停新开借等）以部署合约与审计披露为准。

**Q10：暂停借款时我还能还款吗？**  
A：**暂停只关闭新开借款**；**还款与到期清算仍应可用**（与规则文档 §7 一致）。

**Q11：黑名单是惩罚用户吗？**  
A：黑名单用于统计「非活跃」持仓，参与**可借份额分母**计算（如 LP、交易所托管、`0x…dead` 等），避免这些份额与金库内质押重复扣减；**不是**针对普通用户钱包的「封号」含义。
