## 8. 设计思考（草案）

以下为机制意图与推演，**不等同于**上文的硬规则；实现与盘面可能偏离下列叙述。

1. **金库与外盘解耦**：本币在**其他平台**发行并有自己的交易/LP；金库 BNB 来自**外盘成交产生的本币税**经换币入账（非「LP 手续费」语义）。当前版本博弈对象主要是**税收沉淀的 BNB 可借池**，与外盘定价池仅通过税流耦合。

2. **流通与借款窗口**：借款合约侧 **[3, 6) 天**为**无罚金还款窗口**（见 §4）；**买卖税**（§2）仍按外盘规则收取，二者勿混称「无税」。动态可借、质押会**阶段性**减少可卖流通，是否推价取决于还款、解押与情绪，**非必然**。他人新增质押会改变份额与金库水位，**不保证**单调抬高「后续所有人」的可借上限；叙事上可吸引未借款者参与，但**以当时公式与池子为准**。

3. **复投与杠杆**：借出 BNB 再买本币，相当于**加杠杆**，强度受**金库可借上限、公式、滑点与买卖税**约束，**非无限**；单边行情下可能呈现滚雪球，反转时同样可能连环解押或违约，需双向看待。

4. **价格与还款**：本币下跌时，更易出现**放弃质押、缴罚金或拖过销毁线**，从而**提高销毁与罚金销毁带来的通缩压力**；上涨时，无罚金窗口限制的是**低成本占用 BNB 的时长**，不是外盘盈亏本身。到期还 BNB 常伴随卖压，对外盘参与者**可正可负**。还清后进入下一轮借款/还款博弈。

5. **出发点（目标，非定理）**：买 2% / 卖 3% 略**降低买入摩擦、提高卖出摩擦**；更高卖税主要惩罚**频繁卖出/换手**，与「借 BNB 做多」类行为并存。意图上希望在**窗口内**允许较高复投杠杆叙事，同时用 **3–6 天无罚金期长度**与后续宽限、销毁规则，**试图**缓和「长期免费占用金库 BNB」——能否**平衡**全市场收益取决于实际资金流与行为，**不保证**达成。

6. **抵押与资产负债结构**：以本币为抵押，借出与归还均为 BNB。直觉上存在两条路径——金库 BNB 持续增厚，或本币因税/罚金/销毁而变少——**有人主张**「市场投机与套利会使二者自动趋于某种平衡」。是否成立**有争议**（见 §9.1）。

7. **可借比例 ρ（默认 70%）**与**分母口径**：为金库保留 **1−ρ** 缓冲；**ρ** 及 **§5** 分母是否剔除黑名单质押，均以 **§7** 为准。具体取值**可商榷**，需结合「挤兑式借贷、税流入速度、是否设单地址/全局上限」等再斟酌。**§7 质押分母黑名单**可在链上缓解 LP/所等地址稀释可借份额；文中「假设单地址持仓最多 3%」仍为**推演假设**（见 §9.2）。

8. **开发者回馈**：当前规则未覆盖；设想在**税金换 BNB 入账时**切分一定比例至开发者地址等路径。**争议点**在于：比例、是否多签/锁仓、披露义务，以及与「税进金库」叙事的**信任成本**；亦可对比其他路径（见 §9.3）。

## 9. 待议与争议点（供讨论，非定稿）

以下为社区、产品或实现上**容易产生分歧**之处，便于评审时逐项对齐。

### 9.1 「BNB 增厚 vs 本币变少」是否会自动平衡（对应 §8.6）

- **支持自动平衡的说法**：税持续换 BNB 可补池；本币销毁推高稀缺叙事，可能吸引买盘与交易量，间接反哺税金。
- **反对或限定的说法**：不存在保证收敛的「自动均衡」——可能出现**金库被借空、税入不敷出、币价死亡螺旋、集体策略性违约**等；「投机」既可能稳定池子也可能**加速抽干**。若写进对外文案，建议用**「可能形成反馈」**而非**「会自动平衡」**。

### 9.2 可借 ρ、黑名单与持仓假设（对应 §8.7）

- **ρ（默认 70%）**：调高则资本效率升、池子更易枯竭；调低则反之。可与**单地址借款上限、总未偿上限、冷却时间**等组合替代「一刀切调比例」。
- **质押分母黑名单（§7）**：可缓解 LP、所、销毁地址对可借公式的稀释，但引发**名单治理**争议（谁有权加减、是否误伤正常地址、是否变相中心化）；需与**全链可验证**与 **Timelock** 叙事一并设计。
- **单地址 3%**：若为**链上金库合约内**质押/借款上限，技术上可讨论；若为**外盘持仓**，合约**通常无法直接约束**，需依赖快照、许可名单或链下规则，**争议与成本**较大；与**黑名单**可部分互补，但二者口径不同。
- **公式分母**（已销毁、已质押、是否扣黑名单）在**多笔订单、并发**下的精确定义，仍是实现与审计层面的**经典争议点**。

### 9.3 开发者回馈的路径与取舍（对应 §8.8）

| 路径 | 可能优点 | 常见争议 |
|------|----------|----------|
| 税金换 BNB 后**按比例分流**至开发者 | 简单、可链上验证 | 「税应 100% 归社区/金库」叙事冲突；需公开比例与接收方 |
| **时间锁 / 线性释放** | 降低砸盘与信任质疑 | 实现与治理复杂度上升 |
| **仅对超出某阈值的税或利润**分成 | 缓和与借款人抢池子的观感 | 规则复杂，需清晰定义阈值 |
| **独立融资 / 治理代币**（若未来有） | 与金库池解耦 | 范围超出本文档 |

- 若税金在**外盘合约**收取、再**跨流程**入金库，开发者分成应明确发生在**哪一跳**（本币侧 / BNB 侧 / 入账金库前），避免双重抽成或用户不知情。

### 9.4 其他规则层高频争议（提要）

- **§7 权限边界**：在**不随意提库**前提下，治理侧可能具备 **ρ**、**质押分母黑名单**、**启动/暂停** 等能力；若再叠加 **§9.3 开发者从税金分 BNB**，须逐项约定为**白名单路径**或**入账前分流**，并在链上可验证，避免与「无人可随意转走 BNB」冲突。
- **暂停 vs 用户保护**：长期或选择性暂停**还款**易引发挤兑与舆情；若允许暂停借款但**必须开放还款与清算**，需在合约层写死，而非仅文档承诺。
- **罚金曲线**：提前还最高 90% 是否过激、是否劝退试用；宽限期与正常期**对称爬坡**是否故意为之。
- **≥ 9 天不可还 + 100% 销毁**：公平性、是否存在「非主观逾期」（链拥堵、被黑）；是否保留**极小宽赦或治理申诉**（极敏感）。
- **税 → 换 BNB**：滑点上限、失败重试、MEV；金库 BNB **是否计入「实时」可借**与**待 swap 本币**的会计口径。
- **外盘 LP 与金库**：税基与成交量、刷量、路由绕过税；**两条池子**叙事下用户是否理解「价格在外盘、BNB 在金库」。

## 10. 极端情景与压力测试（非承诺）

以下为**思维实验**：描述在规则不变前提下可能出现的**极端路径**，用于评审与风控讨论；**不构成**对实际结果的预测或担保。

### 10.1 价格极端

- **本币暴涨**：还 BNB 保质押意愿上升；同时 **借 BNB 买币** 可能增多，金库 **BNB 快速被借出**；到期集中卖币还 BNB 时，**卖压 + 卖税** 可能引发剧烈波动。流动性过薄时，可能出现「为还 BNB 而砸盘 → 罚金/违约增多」的反转。
- **本币暴跌**：质押机会成本低，**拖过 9 天、放弃质押** 可能成为理性选择 → **本币销毁上升**，但 **BNB 未必回流金库**；若税入同步萎缩，**可借池长期枯竭**。
- **外盘与金库脱钩**：外盘叙事归零而金库仍有余额时，**借款人 / 金库参与者** 与 **纯外盘持币者** 利益极端分化，社区对立加剧。

### 10.2 金库与税流极端

- **税入骤增**：BNB 池增厚 → **可借上限跳涨**，可能引发抢借与加杠杆潮；若随后税入断档，池子回落，**到期集中压力**放大。
- **税入枯竭**（无成交、绕税、流动性迁走）：新进 BNB 极少 → **池子只减不增**，借款功能名存实亡。
- **税换 BNB 失败或巨滑点**：若「待 swap 本币」与「已入账 BNB」会计不清，**可借额度与用户预期**可能剧烈错位。

### 10.3 行为与协调极端

- **挤兑式借贷**：多人迅速借满 **ρ × 金库总资产**（默认 ρ=0.7，见 §5、§7）→ 后来者借不到，先借者掌握 BNB 流动性，博弈极化。
- **还款时间扎堆**（如卡在第 5～6 天）：Gas 竞争、失败率上升，部分订单**跌入宽限期**，滞纳金上升或出现踩踏式卖币。
- **大规模策略性逾期**：市场认为本币远不及应还 BNB 时，**不还 BNB + 放弃质押** 占优 → 金库 **BNB 净流出**，协议侧以**销毁本币**收场，**BNB 可持续性**完全依赖后续税入。
- **几乎无人借款**：税仍进库则 BNB **沉淀**；叙事上可能被质疑为「死池」或用途不明。

### 10.4 实现与链上极端

- **≥ 9 天边界遇链拥堵**：用户主观想还但交易反复失败，拖过边界 → **不可还且全销毁**，易引发公平性质疑（与 §9.4 呼应）。
- **同区块 / 闪电类操纵**（视最终实现）：若质押与可借份额可被瞬时放大，存在 **单笔占满可借池** 等套利或攻击面，需靠**快照、冷却、上限**等设计逐项排除。
- **MEV 夹击税路 swap**：金库实际到账 BNB 偏少，间接压低 **可借曲线**。

### 10.5 治理与信任极端

- **开发者分成或权限争议**：若被视为「抽干税池」或地址不透明，**借款意愿与币价**可能同时崩塌。
- **黑名单治理**：错误列入正常用户或遗漏 LP/所地址 → **可借公式被操纵**或**稀释问题未解**；与社区「可审计名单」预期冲突。
- **暂停滥用或组合失当**：部署后**长期暂停**、或暂停**还款/清算**导致用户无法退出 → 与 **§7** 信任叙事严重背离；若仅暂停借款而开放还款，亦需防**挤兑式还款**对金库流动性的冲击。
- **密钥 / 多签失守**：与规则无关的毁灭性事件，需在运维层面单独防范。

### 10.6 最坏组合（提要）

| 组合 | 可能现象 |
|------|----------|
| 价崩 + 集体策略违约 | BNB 池难以回补，叙事依赖销毁与后续税 |
| 价暴涨 + 高杠杆 + 到期齐卖 | 流动性断裂，罚金/违约连锁 |
| 税断流 + 池子已大量借出 | 玩法干涸，合约仍运行 |
| 9 天红线 + 非主观链上失败 | 信任与舆情危机 |
| 长期暂停还款或清算 | 用户无法履约/退出，舆情与合规风险陡增 |
