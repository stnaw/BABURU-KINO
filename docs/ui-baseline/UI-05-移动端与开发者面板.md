# 移动端与开发者面板

## 移动端目标

当前移动端不是“桌面缩小版”，而是对局部布局做了多处回落处理，目标是：

- 保住信息主次
- 维持按钮可点性
- 防止局部拥挤和遮挡

## 当前移动端重点适配点

已经有明确适配的部分包括：

- Hero 与导航回落
- 借款区主结构压缩
- 指标卡重新分布
- 开发者面板表单堆叠
- 黑名单列表与借款明细滚动区
- 移动端底部导航 `mobile-dock`

## 开发者面板布局

当前开发者面板在桌面端是双列卡片栅格，移动端会改成单列。

关键规则：

- `developer-grid` 在移动端回落为单列
- `developer-card-wide` 不再跨双列
- `developer-inline-form` 改成单列堆叠

这意味着像“黑名单地址输入 + 加入/移出按钮”这类组合，在移动端会按：

1. 输入框
2. 加入按钮
3. 移出按钮

垂直排列，避免横向重叠。

## 黑名单与借款详情列表

开发者面板中两个高密度区域都已经改为内部滚动：

- 当前黑名单
- 当前链上借款详情

这样面板在移动端不会被长列表无限撑高。

## 当前链上借款详情

当前这块内容包含两层：

- 顶部统计摘要
- 下方订单明细

统计摘要目前包括：

- 汇总订单数
- 提前期订单数
- 免罚期订单数
- 宽限期订单数
- 待清算订单数

## 移动端仍应关注的后续点

虽然当前移动端已具备可用性，但从维护角度看，后续仍建议持续关注：

- 中文长文案断行
- 数字与按钮在超窄屏的拥挤情况
- 开发者面板 owner 操作的触控容错
- 借款列表多选后的底部还款条遮挡关系
