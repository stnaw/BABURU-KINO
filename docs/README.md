# Docs 目录说明

本目录用于沉淀 `BABURU KINKO` 项目的规则、UI、QA、审计和研究资料。  
当前已经按文档用途拆成多个子文件夹，便于后续维护和查找。

## 目录分类

### `docs/audit`

放审计类文档。

主要用途：

- 记录合约或系统层面的审计结论
- 跟踪已确认问题、风险等级和修复建议
- 作为修复与回归验证的依据

当前示例：

- `BaburuKinko-全面审计.md`

### `docs/qa`

放 QA 流程、测试样例、测试任务编排文档。

主要用途：

- 定义测试流程和验证口径
- 沉淀人工 / 自动化测试样例
- 作为 QA 执行和回归的操作基线

当前示例：

- `QA.md`
- `QA-Sample.md`
- `QA-Sample.tasks.yaml`

### `docs/ui-spec`

放前端目标规格文档。

主要用途：

- 描述产品和设计“应该做成什么样”
- 记录页面结构、交互规则、文案口径
- 作为 UI 需求和实现对齐的目标文档

当前示例：

- `前端展示.md`

### `docs/ui-baseline`

放当前前端实现基线文档。

主要用途：

- 描述当前版本前端“实际上已经做成什么样”
- 记录现有布局、视觉语言、刷新策略、移动端例外
- 作为后续 UI 迭代时的现状快照

当前示例：

- `UI-00-当前前端设计基线索引.md`
- `UI-01-页面结构与信息架构.md`
- `UI-02-视觉语言与设计令牌.md`
- `UI-03-关键模块与交互状态.md`
- `UI-04-数据呈现与动态刷新.md`
- `UI-05-移动端与开发者面板.md`

### `docs/tokenomics`

放规则、税率、金库分配和链上口径文档。

主要用途：

- 作为业务规则源
- 统一税率、罚金、分母、金库口径
- 为前端、测试、审计提供共同规则参考

当前示例：

- `税率分配.md`

### `docs/research`

放研究笔记、推演草稿、思考过程文档。

主要用途：

- 保留方案演进过程
- 记录规则推演和设计思考
- 为后续决策提供背景材料

当前示例：

- `税率分配思考.md`

## 使用建议

- 查目标规格：优先看 `docs/ui-spec`
- 查当前 UI 实现：优先看 `docs/ui-baseline`
- 查链上规则口径：优先看 `docs/tokenomics`
- 查测试流程与样例：优先看 `docs/qa`
- 查历史问题与风险：优先看 `docs/audit`
- 查背景思考与草稿：优先看 `docs/research`
