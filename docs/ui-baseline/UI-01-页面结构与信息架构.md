# 页面结构与信息架构

## 页面级结构

当前页面采用单页滚动结构，顶部常驻导航，主体内容按功能分段排列：

1. 顶栏 `topbar`
2. 开发者面板 `developer-panel`
3. 状态横幅 `top-status-banner`
4. Hero 区 `#overview`
5. 指标概览区 `dashboard`
6. 时间线区 `timeline`
7. 借款操作区 `#borrow`
8. 我的借款区 `#loans`
9. 金库手册区 `#faq`
10. 页脚与移动端底部导航

## 顶栏

顶栏由三部分组成：

- 左侧品牌区：Logo + `BABURU KINKO`
- 中间导航：`概览`、`免息借款`、`我的借款`、`金库手册`
- 右侧操作区：连接钱包、开发者入口按钮、语言切换

当前导航更偏“锚点切换”，不是路由页签。点击后滚动到对应区块。

## Hero 区

Hero 区承担品牌和价值主张表达，当前文案为：

- 主标题：`专属于 BABURUSU 的链上无息金库`
- 副标题：`持有 $BABURU，无限次免息提取 BNB —— 随时从 BABURU KINKO 提取`

Hero 下方保留两个主 CTA：

- 进入借款区
- 查看我的借款

## 指标概览区

当前概览区是四卡布局：

1. `可借 BNB`
2. `BABURU`
3. `单笔可借比例`
4. `活跃借款数`

其中前两张卡是信息密度最高的复合卡：

- `可借 BNB` 卡底部附带 `BABURU 质押`、`金库 BNB 总量`
- `BABURU` 卡底部附带 `BABURU 已销毁`、`BABURU 待清算`

## 时间线区

时间线用于把借款生命周期直接可视化，当前展示四段：

1. `1-3 天`
2. `3-6 天`
3. `6-9 天`
4. `≥ 9 天`

每一段都带标签、主文案和说明文案，其中 `3-6 天` 作为高亮安全窗口。

## 借款区

借款区当前是一块主操作面板，内部由两部分组成：

- 左侧输入：质押数量、最小借出比例
- 右侧结果：预估可借 BNB、最小获得、主按钮

当前借款流程是“单按钮状态机”：

- 授权不足：`授权并借款`
- 授权足够：直接借款

## 我的借款区

当前借款列表包含：

- 顶部筛选器
- 全选按钮
- 借款卡片列表
- 底部批量还款条

每张订单卡都显示：

- 当前状态
- 借款时间
- 质押数量
- 借出 BNB

## 金库手册区

金库手册当前使用 FAQ 折叠列表，覆盖：

- BABURU 与金库关系
- 税与金库关系
- 可借额度逻辑
- 罚金与时间窗口
- 批量还款
- 黑名单说明

其中黑名单一项已经从静态文案升级为动态地址列表展示。
