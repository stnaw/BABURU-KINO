# 关键模块与交互状态

## 连接钱包

顶栏主按钮承担钱包连接入口。连接后，页面会切换到已连接状态，并允许：

- 读取钱包 BABURU 余额
- 发起授权
- 发起借款
- 发起还款

错误网络时，会进入显式提示状态。

## 开发者面板

开发者面板仅对 owner 可见。当前包含五类能力：

1. 借款开关
2. ρ 更新
3. 黑名单增减
4. Owner 转移
5. 当前链上借款详情

该面板已经与主页面分离：

- 主页面不再直接展示 `Chain`、`Wallet`、`Test Account`
- 调试信息收敛进开发者面板内部

## 借款面板

借款面板当前的状态机包括：

- 默认待输入
- 授权中
- 借款中
- 成功
- 失败
- 网络错误
- 最小质押不足

相关状态通过：

- 面板内状态文案
- 浮动 toast
- 按钮文本切换

共同表达。

## 借款时间窗口

当前借款窗口区块承担两层作用：

- 规则教育
- 用户心智引导

当前口径中，`3-6 天` 是“最佳还款时间”，提前和延后都强调罚金，`≥ 9 天` 则进入不可还款、待清算路径。

## 我的借款

借款列表支持：

- 全部 / 提前期 / 正常期 / 宽限期 / 已超期 筛选
- 多选
- 底部批量还款条

对于已超期订单，当前 UI 逻辑是：

- 不作为普通可还款项
- 保留状态展示
- 通过文案强调等待清算

## 金库手册

FAQ 当前不是一个弱化角落模块，而是主页面正式区块。它承担了产品教育、链上逻辑翻译和黑名单透明度展示三种职责。

当前 FAQ 已与真实数据发生连接：

- 金库黑名单地址列表是动态的
- 会合并默认逻辑黑名单与链上黑名单

## 全局反馈

当前全局反馈机制由 `floating-toast` 承担，主要用于：

- 借款状态反馈
- 还款状态反馈
- 开发者操作反馈
- 网络与权限警告

这意味着页面已经形成统一反馈通道，而不是每个模块自己长一套消息样式。
